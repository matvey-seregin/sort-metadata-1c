Сортировка объектов метаданных

Обработка предназначена для приведение конфигурации к требованиям 1С:Совместимо в части порядка расположения объектов метаданных в дереве конфигурации:

  • Объекты конфигурации поставщика должны быть упорядочены по алфавиту;
  • Объекты с префиксом удалить должны быть в конце списка;
  • Объекты дополняющие конфигурацию поставщика должны быть выделены отдельным префиксом, быть отсортированными по алфавиту, располагаться после объектов конфигурации поставщика.

Запуск обработки

Обработка запускается в пользовательском режиме в информационное базе, объекты которой необходимо упорядочить в тонком или толстом клиенте в управляемом приложении.

Если инфомрационная база подключена к хранилищу необходимо захватить корень конфигурации.

Если информационная база на поддержке необходимо разрешить изменения корня конфигурации.

Настройки префиксов

Настройка префиксов осуществляется после чтения информации о метаданных из информационной базы.

Использование - если флажок не стоит, то префикс будет игнорироваться при сортировке.

Порядок - число - по умолчанию для всех префиксов 1. Если объекты с каким-то префиксов должны быть после остальных можно установить значение упорядочивания более 1. Если объекты с каким-то префиксом должны быть в начале списка (перед объектами конфигурации поставщика) им необходимо установить отрицательное значение упорядочивания. Для объектов без префикса значение упорядочивания считается равным 0.

Настройки префиксов сохраняются между запусками обработки в одной информационной базе.

Принцип действия обработки

  1. Выполняется пакетный запуск конфигуратора для выгрузки файлов в XML;
  2. Загружаются возможные префиксы объектов метаданных;
  3. Пользователю предоставляется возможность указать настройки префиксов (какие используются, в каком порядке необходимо сортировать);
  4. Модифицируется файл корня Configuration.xml;
  5. Модифицированный файл корня подгружается в информационную базу с помощью частичной загрузки файлов из XML.
